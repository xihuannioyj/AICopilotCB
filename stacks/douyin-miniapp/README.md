# douyin-miniapp

## 技术栈定位

- 适用范围：抖音小程序项目及其配套协同规则
- 继承关系：默认继承 `core/` 的任务闭环、回流协议和收尾模板

## 推荐读取顺序

1. 先读 `core/README.md`
2. 再读 `docs/requirements/02-技术栈适配层.md`
3. 最后结合项目覆盖层确认平台接口、内容能力和提审路径

## 最小入口

- 构建入口：优先识别平台配置文件、应用入口配置与项目脚本
- 运行入口：优先识别页面入口、本地预览路径和平台能力初始化位置
- 测试入口：优先识别页面验证路径、接口联调路径与提审前检查项

## 常见角色组合

- architect + frontend/app + backend
- test-collector + defect-triage + backend

## 常见风险与交接

- 需先确认平台授权、内容能力和登录链路限制
- 不同平台能力与微信小程序不完全等价，不能直接照搬
- 交接时要写清页面入口、平台差异点和提审前验证范围
