# projects/livehome

## 目录定位

- machine_readable_tag: aicopilotcb-livehome-overlay-readme-v1
- 目录角色：当前样板项目的项目覆盖层示例

## 与底座的关系

- 继承 `core/` 的通用协作协议
- 继承 `stacks/` 的技术栈执行差异说明
- 只在本目录承接 LiveHome 样板自己的目录、脚本、入口和例外约束

## 应放在覆盖层的内容

- 某个项目自己的业务目录边界
- 该项目独有的任务板、派发索引和运行态约束
- 只对单一项目成立的接口规范、权限规则、初始化要求和入口闭环约束
- 仅对特定研发流程成立的 Prompt、Agent 约束或守卫要求

## 最小说明要求

每个项目覆盖层文档至少要说明：

1. 项目根目录和主要业务根目录
2. 项目专属任务板与 runtime 路径
3. 项目专属脚本、Prompt、守卫或例外约束
4. 哪些规则可上移，哪些规则必须继续留在覆盖层

## 维护原则

- 继续通用化时，默认先把强项目绑定规则留在覆盖层，再逐步识别哪些部分可以继续上移到 `core/` 或 `stacks/`。
- 新项目接入时，应新建 `projects/{project-name}/`，而不是直接修改本样板目录。
