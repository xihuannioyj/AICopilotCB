# 初始化配置与运行时上下文说明

## 文档定位

- machine_readable_tag: aicopilotcb-project-context-guide-v1
- 文档角色：说明工作区初始化配置、角色边界配置与运行时上下文文件的字段含义和使用顺序
- 适用对象：AICopilotCB 演进治理角色、Prompt 维护者、Agent 维护者、后续接入新项目的 Owner

## 三份配置文件

当前项目上下文采用三份文件协作：

1. `.github/project-context/workspace-init.json`
2. `.github/project-context/role-boundaries.json`
3. `.github/project-context/active-context.json`

它们分别负责：

- `workspace-init.json`：定义工作区级默认值。
- `role-boundaries.json`：定义角色级边界和读取要求。
- `active-context.json`：定义运行时激活项目、实例切换和会话级覆盖项。

## 解析优先级

默认解析顺序如下：

1. `user-explicit-target`
2. `active-context-overrides`
3. `role-boundaries`
4. `workspace-init`

这意味着：

- 用户当前任务里明确指定的目标优先级最高。
- 运行时激活实例和覆盖项优先于角色默认边界。
- 角色默认边界优先于工作区默认值。
- 工作区默认值只作为兜底，不直接替代角色边界。

## workspace-init.json 字段说明

### 顶层字段

- `schemaVersion`：配置结构版本。
- `workspaceId`：工作区唯一标识。
- `workspaceName`：工作区展示名称。
- `defaultProject`：默认项目名。
- `runtimeContextFile`：运行时上下文文件路径。
- `description`：本文件用途说明。

### projects.{projectName}

- `root`：项目根路径。
- `projectType`：项目类型，例如控制面、业务工程、样板工程。
- `languages`：该项目默认涉及的语言列表。
- `frameworks`：该项目默认涉及的框架或协作基础设施。
- `defaultWriteRoots`：项目级默认可写范围。
- `defaultForbiddenWriteRoots`：项目级默认禁止写入范围。
- `requiredReads`：默认必须先读取的文档和脚本。
- `validation.changed`：校验全部变更的默认命令。
- `validation.markdown`：校验单个 Markdown 的默认命令模板。
- `validation.files`：校验普通文件的默认命令模板。

## role-boundaries.json 字段说明

### 顶层字段

- `schemaVersion`：配置结构版本。
- `defaultProject`：角色未显式指定项目时的默认项目。
- `runtimeContextFile`：运行时上下文文件路径。
- `resolutionOrder`：角色边界解析优先级。

### agents.{agentName}

- `project`：该角色默认归属的项目。
- `roleType`：角色分类，用于区分治理、交付、采证、分诊等模式。
- `mode`：角色运行模式，例如 `control-plane-only` 或 `owner-root-only`。
- `supportsRuntimeContext`：是否允许运行时上下文覆盖默认边界。
- `allowedWriteRoots`：该角色默认允许写入的目录范围。
- `forbiddenWriteRoots`：该角色默认禁止写入的目录范围。
- `requiredReads`：该角色开始执行前的必读文件。
- `handoffTargets`：该角色默认推荐交接对象。

## active-context.json 字段说明

### 顶层字段

- `schemaVersion`：配置结构版本。
- `activeWorkspace`：当前激活工作区。
- `activeProject`：当前激活项目。
- `activeProfile`：当前激活配置档。
- `activeAgent`：当前主用或优先治理的角色。
- `description`：本文件用途说明。
- `resolutionOrder`：运行时上下文参与解析后的优先级说明。

### overrides

- `project`：临时切换项目实例时使用的项目覆盖项。
- `writeRoots`：临时补充或收紧可写范围。
- `forbiddenWriteRoots`：临时补充禁止写入范围。
- `requiredReads`：临时追加必读文件。
- `validation`：临时覆盖校验命令。

### session

- `useWorkspaceDefaults`：是否启用工作区默认值。
- `useRoleBoundaries`：是否启用角色边界默认值。
- `allowOverrideByUserIntent`：是否允许被用户本轮目标覆盖。

## 维护原则

- 需要改工作区级默认值时，优先改 `workspace-init.json`。
- 需要改角色边界时，优先改 `role-boundaries.json`。
- 需要切换当前激活实例或临时覆盖项时，优先改 `active-context.json`。
- 不要把工作区实例参数直接散写回 Agent 文本，避免多份配置漂移。

## 推荐工作流

1. 新会话先读 `workspace-init.json`。
2. 再读 `role-boundaries.json`。
3. 如存在运行时实例切换，再读 `active-context.json`。
4. 然后进入 `.github/copilot-instructions.md` 和对应 Prompt / Agent 规则。
5. 最后根据当前任务决定是否需要再读任务文档、样板目录或业务契约。
