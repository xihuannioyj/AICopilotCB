# 05-Prompt与工作流入口

## 文档定位

- machine_readable_tag: aicopilotcb-req-prompts-v2
- 评审对象：AICopilotCB Prompt 与工作流入口需求细则
- 文档角色：定义新会话起手、任务分流和专用工作流入口

## 背景与问题

协同成本经常不是出在执行本身，而是出在入口选错。若没有稳定的 Prompt 入口体系，同一个任务会在方案维护、业务开发、盘点汇报之间来回切换，最终造成范围漂移、改错文件、误用模板或错误读取任务状态。

## 建设目标

- 让不同任务在开始阶段就进入正确执行流。
- 让 Prompt 入口成为稳定路由器，而不是说明性文档。
- 降低先做再纠偏的返工成本。

## 范围

本主题覆盖：

- 新会话起手入口。
- 协同方案维护入口。
- 业务需求启动入口。
- readiness、值班、汇报与阶段验收入口。
- 入口冲突时的优先级规则。

## 非目标

- 不在本主题承载具体业务实现细节。
- 不替代 Agent 角色文档说明职责边界。
- 不要求一个 Prompt 同时解决多个主流程。

## 核心需求

### 1. Prompt 入口必须覆盖主要任务类型

至少需要稳定支持以下四类入口：

1. Workspace 起手入口：用于新开会话先读取仓库规则并完成分流。
2. 协同方案维护入口：用于 Prompt、Agent、Instructions、Hook、Runbook、模板和控制面脚本维护。
3. 业务需求启动入口：用于多 Agent 拆任务、派发、开发、联调与收尾。
4. Readiness / 值班 / 汇报入口：用于盘点状态、形成管理视图和阶段结论。

### 2. 同一任务只能命中一个主入口

- 不允许多个主入口并行启动后再靠口头修正。
- 若任务同时含方案维护与 readiness 盘点，必须先维护，再盘点。
- 若范围不清，优先走起手入口完成分流，而不是直接进入专用 Prompt。

### 3. 每个 Prompt 都必须说明适用范围与非适用范围

- 适用场景必须清楚到可以直接判断该不该用。
- 非适用场景必须写明，避免被误用为万能入口。
- Prompt 之间允许引用，但不允许重复定义冲突规则。

### 4. Prompt 入口必须与导航文档一致

- 入口清单、说明书、导航文档和控制面指令必须使用同一套分类口径。
- 若任一入口规则改变，导航文档必须同步更新。

## 建议交付物

- 稳定的 Prompt 清单。
- 每个 Prompt 的适用范围与非适用范围说明。
- 典型任务与入口映射表。
- 面向 Owner 的导航入口说明。

## 验收标准

1. 用户面对一个任务时，能够明确判断应从哪个入口开始。
2. 不同 Prompt 之间的职责边界清楚，冲突时有优先级规则。
3. 协同方案维护不会误入业务实现链路。
4. 新会话起手后能快速建立上下文，而不是再次手动找规则。

## 风险与约束

- 若 Prompt 过多且边界不清，分流成本会重新上升。
- 若导航与 Prompt 口径不一致，用户会回到靠记忆找入口的状态。
- 若维护入口默认进入业务代码，协同方案治理任务会持续污染业务根目录。
