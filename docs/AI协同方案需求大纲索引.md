# AI协同方案需求大纲索引

## 文档定位

- machine_readable_tag: aicopilotcb-requirements-index-v3
- 文档角色：AICopilotCB 需求体系唯一导航入口
- 使用方式：先读本索引，再按主题进入对应专题文档

## 使用说明

- 本索引是 AICopilotCB docs 目录保留的唯一总入口。
- 需求总目标、拆分结构、自进化约束与演进路线，均已下沉到各专题 requirements 中。
- 工作区初始化配置与运行时上下文说明，保留在 docs 根目录，由本索引统一登记和分发。
- 评审时不再依赖单独的总文档或拆分方案文档。

## 大纲索引

- 对应文档：docs/requirements/01-通用核心层.md

### 02. 技术栈适配层

- 对应文档：docs/requirements/02-技术栈适配层.md

- 典型内容：构建测试约束、目录约束、联调方式、技术栈专属模板

### 03. 项目覆盖层

### 04. Agent与角色协同

### 05. Prompt与工作流入口

## 推荐阅读顺序

1. 先读 01、02、03，理解三层结构和分层边界。
2. 再读 04、05、06，理解角色协同、Prompt 分流和 Markdown 写入安全。
3. 然后读 07、08，理解运行时编排、模板体系和新项目接入路径。
4. 最后读 09，理解三份 project-context 配置文件如何驱动 Agent 初始化与运行时切换。
5. 若目标是落地某个具体项目，再补读 projects/ 与 stacks/ 下的对应说明。

## 维护约束

- 新增需求主题时，先更新本索引，再新增对应专题文档。
- 若专题文档重命名、拆分或合并，必须同步回写本索引。
- 本索引只负责导航，不替代专题文档承载完整细则。
- docs 根目录允许保留已在本索引登记的运行说明类文档；未登记的新文档不得散落新增。
