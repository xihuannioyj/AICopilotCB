# AUTO_DEV 完全可用检查清单

## 文档定位

- machine_readable_tag: aicopilotcb-auto-dev-readiness-checklist-v1
- 文档角色：定义控制面进入“完全可用”前的最小检查项

## 检查项

1. Prompt 入口清单存在，且主入口分流无冲突。
2. Agent 角色边界存在，且默认边界以配置真源驱动。
3. 受保护 Markdown 写入协议可执行，并可通过 guard 校验。
4. tasks 主看板、候选池、治理表与 runtime 主数据均已建立。
5. 至少一套值班/汇报/演练导航与模板存在。
6. 新项目接入最小步骤已文档化，不依赖口头说明。
7. 当前仓库中的关键导航路径不存在断链引用。

## 判定规则

- 以上任一项未满足，都不得判定为“完全可用”。
- 若只有占位数据而无真实运行结果，只能判定为“控制面骨架已就绪”。
- 若 guard 校验无法通过，只能判定为“不可交付”。
