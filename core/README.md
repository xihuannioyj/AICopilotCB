# core

## 目录定位

- machine_readable_tag: aicopilotcb-core-readme-v1
- 目录角色：承载项目无关、技术栈无关的 AI 协同核心规则

## 适用范围

- 角色边界定义方法
- 任务入口、执行、验证、回流、收尾五段式闭环
- 统一收尾模板与最小验收口径
- Markdown 安全写入与守卫校验闭环
- Prompt 分流和控制面通用组织原则

## 不适用范围

- 具体技术栈的构建、运行、测试命令
- 具体项目的目录、脚本、权限 key、接口前缀和例外约束
- 任何单项目专属的业务规则

## 推荐读取顺序

1. 先读 `docs/requirements/01-通用核心层.md`，确认核心层需求边界。
2. 再读 `.github/copilot-instructions.md` 中已经稳定落地的全局协议。
3. 最后回看 `stacks/` 和 `projects/`，判断哪些规则仍应留在适配层或覆盖层。

## 当前应沉淀的内容

- Agent 职责边界的定义方式
- 任务拆分、交接、回流和完成判定协议
- Markdown 安全写入与校验闭环
- Prompt 组织原则
- 协同运行手册的公共骨架

## 维护原则

- 先验证规则可复用，再上移到 core。
- 一旦规则写死项目路径或项目术语，应立即回退到 `projects/` 或 `stacks/`。
- core 中的规则必须能在更换项目名与技术栈后仍然成立。
